Библиотека для выделения границ на изображениях. Содержит классические методы (Sobel, Prewitt, Canny) и **авторский метод векторного освещения**, разработанный для эффективной детекции границ на цветных изображениях.

## ✨ Особенности

-  **Цветочувствительность** – метод `vector_lighting` выявляет **изолюминантные границы** (переходы между разными цветами с одинаковой яркостью), которые не видны классическим методам.
-  **Конкурентное качество** – средний F1-score **0.788** на синтетическом переборе из 20 736 конфигураций и **0.740** на 6-картиночной таблице бенчмарка ниже (сопоставим с Canny, на цветовых тестах превосходит на 88–210%).
-  **Производительность** – векторизованная реализация на NumPy/SciPy. **Быстрее Canny** при сопоставимом качестве.
-  **Научно обоснован**: Параметры оптимизированы на основе полного перебора **20 736 конфигураций**.
-  **Гибкость** – настраиваемые режимы освещения, перестановки каналов, методы слияния.

## ⚙️ Параметры метода `vector_lighting`

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|:------------:|----------|
| `mode` | `int` | `3` | Режим освещения: `0`=1 вектор, `1`=2, `2`=4, `3`=8 векторов. Больше векторов → выше качество, медленнее. |
| `sigma` | `float` | `1.0` | Гауссово сглаживание. Уменьшает шум, но может размывать тонкие границы. |
| `binary_percentile` | `float` | `0.05` | Процент пикселей для бинаризации. `0.05` = оставить 5% самых ярких. `0.0` = порог по среднему. ⚠️ На градиентах используйте `0.0`. |
| `use_permutations` | `bool` | `True` | Перебирать перестановки каналов. Даёт прирост +1.7%, но замедляет в 6 раз. |
| `merge_method` | `str` | `'max'` | Метод слияния: `'mean'`, `'max'`, `'adaptive_mean'`, `'weighted'`. `'max'` сохраняет сильнейший отклик. |
| `threshold_method` | `str` | `'percentile'` | Метод порога: `'mean_std'`, `'median'`, `'percentile'`. |
| `threshold_factor` | `float` | `0.25` | Множитель для `'mean_std'`. Игнорируется при `'percentile'`. |
| `height_weight` | `float` | `1.0` | Вес влияния высоты (третьего канала) на отклик. |
| `clean_noise` | `bool` | `False` | Морфологическая очистка. ⚠️ Ухудшает F1 на 42%, удаляет тонкие границы. **Не рекомендуется**. |
| `channel_roles` | `tuple` | `None` | Явные роли каналов `(x, y, z)`. Игнорирует `use_permutations`. |
| `binary` | `bool` | `True` | Применять бинаризацию. Если `False`, возвращает карту градиентов. |
| `return_debug` | `bool` | `False` | Вернуть dict с отладочной информацией. |

## 🎯 Рекомендации по настройке

### Для максимального качества (параметры по умолчанию)

Используйте конфигурацию, полученную в результате полного перебора **20 736 вариантов**:

```python
from vector_lighting import vector_lighting

edges = vector_lighting(
    image,
    mode=3,                # 8 векторов освещения
    sigma=1.0,             # Оптимальное сглаживание
    binary_percentile=0.05,# Оставлять 5% пикселей
    use_permutations=True, # Все перестановки каналов
    merge_method='max',    # Лучший метод слияния
    threshold_method='percentile',
    threshold_factor=0.25,
    height_weight=1.0,
    clean_noise=False      # Сохранять тонкие границы
)
```
Результат: Mean F1 = **0.788** на синтетическом переборе из 20 736 конфигураций (против **0.740** на 6-картиночной таблице бенчмарка выше).

### Для высокой скорости

Отключите перестановки и уменьшите количество векторов:

```python
edges = vector_lighting(
    image,
    use_permutations=False, # Отключить перестановки
    mode=2                  # 4 вектора вместо 8
)
```
